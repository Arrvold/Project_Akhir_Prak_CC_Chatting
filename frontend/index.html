API Documentation

Documentation for User, Contact, and Chat API

UserController API

1. Register

Endpoint: POST /api/register

Description: Register a new user with username, email, nickname, and password.

Request Body:

{
  "username": "string",
  "password": "string",
  "email": "string",
  "nickname": "string"
}

Response:

Status 201:

{
  "message": "Registrasi berhasil",
  "user": {
    "id_user": "int",
    "username": "string",
    "email": "string",
    "nickname": "string"
  }
}

2. Login

Endpoint: POST /api/login

Description: Login a user with username and password to get an access token.

Request Body:

{
  "username": "string",
  "password": "string"
}

Response:

Status 200:

{
  "message": "Login berhasil",
  "accessToken": "string",
  "user": {
    "id_user": "int",
    "username": "string",
    "email": "string",
    "nickname": "string"
  }
}

3. Refresh Token

Endpoint: POST /api/refresh-token

Description: Get a new access token using a valid refresh token.

Response:

Status 200:

{
  "accessToken": "string"
}

4. Logout

Endpoint: POST /api/logout

Description: Logout a user and clear the refresh token.

Response:

Status 200:

{
  "message": "Logout berhasil"
}

5. Update User

Endpoint: PUT /api/user/:id

Description: Update user information such as username, email, nickname, or password.

Request Body:

{
  "username": "string",
  "email": "string",
  "nickname": "string",
  "password": "string"
}

Response:

Status 200:

{
  "message": "User berhasil diperbarui"
}

6. Delete User

Endpoint: DELETE /api/user/:id

Description: Delete a user based on the given ID.

Response:

Status 200:

{
  "message": "User berhasil dihapus"
}
